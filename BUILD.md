# Build Instructions

## Development Setup

1. Install dependencies:

```bash
npm install
```

2. Run development server (demo app):

```bash
npm run dev
```

3. Build the library:

```bash
npm run build
```

4. Type check:

```bash
npm run type-check
```

## Build Output

The build process creates:

- `dist/vue-apexgantt.es.js` - ES module build
- `dist/vue-apexgantt.umd.js` - UMD build
- `dist/index.d.ts` - TypeScript declarations (auto-generated by vite-plugin-dts)
- Additional `.d.ts` files for each component/module

## Publishing to NPM

1. Update version in package.json
2. Build the library: `npm run build`
3. Publish: `npm publish`

## Key Features

1. **Vue 3 Composition API**: Modern Vue 3 with `<script setup>` syntax
2. **TypeScript**: Full type safety and IDE support
3. **Event System**: Vue events emitted from native CustomEvents
4. **Reactive**: Watches prop changes and updates chart automatically
5. **Template Refs**: Expose methods via ref for programmatic control
6. **Composables**: useGanttData for data parsing
7. **License Management**: Easy license key setup
8. **Demo App**: Comprehensive examples included

## Notes

- The component uses Vue 3 Composition API exclusively (no Options API)
- Events from ApexGantt's CustomEvents are re-emitted as Vue events
- The component is reactive to prop changes (tasks, viewMode, theme, options)
- Template ref methods allow programmatic control (zoom, update, etc.)
- The demo app is configured to run with `npm run dev` and uses the local src/ files
- For production, the library should be built with `npm run build` before publishing
- TypeScript declarations are automatically generated using `vite-plugin-dts` during the build process
- The build system uses Vite for optimal bundling and tree-shaking
